Was ist OpensourceBackup?

OpensourceBackup sichert Dateien, Ordner und Datenbanken auf deinen Servern und Clients — zentral verwaltet, automatisch überwacht.

Control Plane (zentral)
  → verwaltet Systeme, Policies, Jobs, Snapshots
  → plant Backups (Cron) + überwacht (Dead-Man's-Switch)

Agent (auf jedem Zielsystem)
  → holt Jobs → führt restic backup aus → meldet zurück

Was du sichern kannst: Dateien & Ordner, PostgreSQL (pgBackRest), komplette Systeme (Restic), Kubernetes (Velero).

🔥 Installation auf Proxmox (Empfehlung)

Empfohlener Weg: Debian 12 LXC Container auf Proxmox VE.

Schritt 1 — LXC Container anlegen

# Im Proxmox Shell als root:
pveam update
pveam download local debian-12-standard_12.7-1_amd64.tar.zst

pct create 200 local:vztmpl/debian-12-standard_12.7-1_amd64.tar.zst \
  --hostname opensourcebackup \
  --memory 2048 --cores 2 \
  --rootfs local-lvm:20 \
  --net0 name=eth0,bridge=vmbr0,ip=dhcp \
  --features nesting=1 --unprivileged 1

pct start 200 && pct enter 200

Schritt 2 — Install Script

Im Container als root:

curl -fsSL \
  https://raw.githubusercontent.com/cerberus8484/opensourcebackup/main/scripts/install-server.sh \
  | bash

Das Script installiert Docker, PostgreSQL 16, Redis 7, führt Migrationen aus und erstellt einen systemd-Service.

Schritt 3 — Windows Agent installieren

# Token aus der Web-UI (Agents → Enroll Agent)
$env:CONTROL_PLANE_URL  = "http://192.168.1.xxx:8080"
$env:ENROLLMENT_TOKEN   = "<token>"
$env:RESTIC_PASSWORD    = "<passwort>"
$env:RESTIC_REPO        = "Z:\OpenSourceBackup"
.\opensourcebackup-agent.exe

Schritt 4 — Linux Agent installieren

CONTROL_PLANE_URL=http://192.168.1.xxx:8080 \
ENROLLMENT_TOKEN=<token> \
RESTIC_PASSWORD=<passwort> \
RESTIC_REPO=/mnt/nas/backups \
bash <(curl -fsSL http://192.168.1.xxx:8080/scripts/install-agent.sh)

Verwaltungs-Befehle

systemctl status opensourcebackup       # Status
journalctl -u opensourcebackup -f      # Logs
systemctl restart opensourcebackup     # Neustart
docker compose -f /opt/opensourcebackup/docker-compose.yml ps  # DB-Status
💡 Alternativ direkt auf dem Proxmox-Host ausführen (nicht empfohlen für Produktion).

Lokaler Start (Entwicklung)

make dev-up && make migrate-up && make run
curl http://localhost:8080/health  # → {"status":"ok"}

Agent auf einem Zielsystem einrichten

1. System anlegen
curl -X POST http://localhost:8080/v1/systems \
  -H "Content-Type: application/json" \
  -d '{"Hostname":"mein-server","RiskClass":"standard"}'
# → {"ID":"uuid-des-systems",...}
2. Enrollment-Token erzeugen (gilt 30 Min)
curl -X POST http://localhost:8080/v1/systems/{uuid}/enrollment-token
# → {"token":"Xk3mNp...","expires_at":"..."}
3. Agent starten — enrollt sich automatisch
CONTROL_PLANE_URL=http://localhost:8080 \
ENROLLMENT_TOKEN=Xk3mNp... \
RESTIC_PASSWORD=geheimes-passwort \
RESTIC_REPO=s3:mein-bucket/backups/mein-server \
./agent
# Token wird in data/agent-token gespeichert
# Agent pollt ab jetzt alle 30s nach Jobs

Beim nächsten Start genügt AGENT_TOKEN_FILE=data/agent-token — kein Enrollment-Token mehr nötig.

Repositories /v1/repositories

curl -X POST http://localhost:8080/v1/repositories \
  -d '{"Type":"restic","Location":"s3:bucket/backups","ObjectLockEnabled":true}'
FeldPflichtWerte
Type✅restic, borg, pgbackrest, velero
Location✅s3:bucket, sftp:host:/path, /local/path
ObjectLockEnabled—WORM-Schutz gegen Ransomware

Policies /v1/policies

curl -X POST http://localhost:8080/v1/policies \
  -d '{
    "Name": "nightly",
    "Engine": "restic",
    "RepositoryID": "uuid-des-repositories",
    "Includes": ["/home","/etc"],
    "Excludes": ["/home/*/.cache"],
    "Schedule": "0 2 * * *",
    "Retention": {"daily":7,"weekly":4,"monthly":12}
  }'
⚠️ RepositoryID muss gesetzt sein — sonst schlägt jeder Backup-Job fehl.
ScheduleBedeutung
0 2 * * *Täglich 02:00 Uhr
0 */6 * * *Alle 6 Stunden
0 2 * * 0Wöchentlich, Sonntags

Snapshots /v1/snapshots

GET /v1/snapshots         alle Snapshots

Snapshots werden automatisch vom Agent beim complete-Aufruf registriert — mit repository_id aus der Policy.

Scheduler & Dead-Man's-Switch

Cron-Jobs werden automatisch als pending-Jobs erstellt. Alle 5 Minuten prüft der Dead-Man's-Switch ob Jobs termingerecht gelaufen sind:

{"level":"WARN","msg":"dead-man: overdue job detected","policy_name":"nightly"}

FAQ

Agent meldet "re-enrollment required"?

Token wurde revoked. Neuen Enrollment-Token erstellen und Agent neu starten mit ENROLLMENT_TOKEN=....

Backup schlägt fehl: "policy has no repository"?

curl -X PUT /v1/policies/{id} -d '{"RepositoryID":"uuid-des-repositories",...}'

Alles zurücksetzen:

make migrate-down && make migrate-up

Sicherheitshinweise

  • Agent-Token in data/agent-token mit Rechten 0600
  • Enrollment-Token gilt nur 30 Minuten und nur einmal
  • RESTIC_PASSWORD und DATABASE_URL niemals in Logs
  • Jede Response enthält automatisch Security Headers (CSP, HSTS, X-Frame, …)
  • Agent sieht nur Jobs des eigenen Systems — kein Cross-System-Zugriff möglich
