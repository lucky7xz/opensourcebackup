1. Einführung und Ziele

OpensourceBackup ist eine Backup Control Plane — kein neues Backup-Tool, sondern eine Plattform zur Orchestrierung bewährter Backup-Engines (Restic, Borg, pgBackRest, Velero) über 100+ Systeme.

PrioritätZiel
1Zentrales Management von Backup-Jobs für heterogene Systeme
2Automatische Verifikation der Restore-Integrität
3Dead-Man's-Switch: Alert wenn erwartete Jobs ausbleiben
4Betrieb auf Kubernetes und Docker Compose
QualitätsmerkmalBeschreibung
KorrektheitBackup-Daten zuverlässig verarbeiten
SicherheitmTLS, RBAC, Security Headers, keine Credentials im Code
WandelbarkeitCCD-Prinzipien, Interface-basiert, testgetrieben
BetriebsbereitschaftGraceful Shutdown, Health-Endpoint, Middleware-Chain

6. Laufzeitsicht

API-Request

HTTP Request
  → Recovery        (panic-safe)
  → SecurityHeaders (6 HTTP Security Headers)
  → BodyLimit 1MB   (413 bei Überschreitung)
  → Logging         (method, path, status, duration_ms)
  → Timeout 30s     (503 bei Überschreitung)
  → Handler → Store → PostgreSQL
  ← JSON Response

Dead-Man's-Switch

Ticker alle 5 Minuten
  → für jede Policy mit Schedule:
      → cronInterval(schedule) → erwartetes Intervall
      → JobStore.LatestByPolicyID()
      → wenn last_job älter als Intervall × 1.5:
          → slog.Warn("dead-man: overdue job detected")
