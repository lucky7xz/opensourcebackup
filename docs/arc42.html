Arc42 Architektur
OpensourceBackup — Stand B1–B9.7

1. Einführung und Ziele

OpensourceBackup orchestriert Restic, Borg, pgBackRest und Velero über 100+ Systeme. Der Agent läuft auf Zielsystemen, führt Backups aus und kommuniziert sicher über Bearer-Token-Authentifizierung.

PrioritätZiel
1Zentrales Management für heterogene Systeme
2Automatische Restore-Verifikation
3Dead-Man's-Switch bei ausbleibenden Jobs
4Sicherer Agent-Flow: Enrollment → Bearer Token → geschützte Routen

6. Laufzeitsicht

Enrollment

Admin: POST /v1/systems/{id}/enrollment-token
  → GenerateToken() → HashToken() → DB (TTL 30 Min)
  → raw Token einmal zurückgeben (nie loggen)

Agent: POST /v1/agent/enroll {enrollment_token}
  → Hash prüfen: nicht abgelaufen, nicht benutzt
  → used_at setzen
  → neuen Agent-Token erzeugen → Hash speichern
  → in data/agent-token (0600) speichern

Gesicherter Backup-Flow

Cron → BackupJob{pending} in DB

Agent pollt GET /v1/agent/jobs + Bearer Token
  → AgentAuth: HashToken → ValidateAndTouch → system_id
  → nur eigene Jobs sichtbar

PUT /v1/agent/jobs/{id}/start
restic backup --json
PUT /v1/agent/jobs/{id}/complete {snapshot_id, bytes, paths}
  → Job=success + Snapshot mit policy.repository_id

7. Verteilungssicht

Entwicklung:
  make dev-up + make migrate-up + make run → Control Plane :8080
  ENROLLMENT_TOKEN=... make run-agent → Agent auf Zielsystem

Produktion:
  Kubernetes: Ingress (TLS) → control-plane Pods → PostgreSQL
  Agent → Bearer Token → Ingress
